# INSA Doctrine

## 1. Vibe Done
Vibe coding creates artifacts. Vibe done admits them.
Done is not "generated", "compiled", or "looks right".
**Done = evidence, not confidence.**
`Done = closed field + lawful motion + bounded delta + canonical evidence + deterministic replay.`

## 2. Byte Law & Byte-Width Semantic Multiplexing
Speed is ontological, not an optimization pass. The ontology is byte-shaped so the machine can run it cheaply.
8 is not arbitrary bounded cognition. 8 is byte-width semantic multiplexing. A single byte can carry 256 activation surfaces.

## 3. Admitted vs Unadmitted
The boundary is not stable Rust vs. nightly Rust. The boundary is admitted vs unadmitted.
Nightly, unsafe, SIMD, intrinsics, and const machinery are all candidates.
Unproven stable code is experimental. Proven nightly code is production. Truthforge decides.
`Admitted = exact law + exact layout + exact encoding + exact target + exact benchmark + exact failure + revocable evidence.`

## 4. The TCPS (Toyota Code Production System)
INSA is Toyota for executable law.
Preventing wrong work from forming is the core value.
Stop the line on invalid states, missing evidence, unproofed routes, or layout drift.
